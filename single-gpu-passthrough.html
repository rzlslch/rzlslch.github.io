Preambule

Kalo mau skip ke teknis nya, langsung kesini technicality

Dalam menggunakan sistem operasi, sebenarnya kita punya banyak pilihan. Windows, Macos, Linux, dll. Saya sendiri pake linux. Semua kebutuhan berkomputer saya sudah tercukupi. Tapi ada satu case yang beneran saya butuh (pengen) Windows. Dan ini spesifik sekali.

Di linux, hmmm… Sampai sekarang saya masih belum bisa buat bikin mod ini berjalan dengan baik di linux. Jalan mod nya, ga ctd, tapi fungsionalitas mod nya jadi ngaco.

Saya butuh (pengen) Windows (yang mana nantinya ternyata saya udah males maen game, tapi itu cerita buat lain waktu). Di komputer saya, udah terinstall linux. Buat orang yang butuh pakai dua atau lebih sistem operasi, ada beberapa cara yang bisa ditempuh. Coba kita aplikasikan.

Sebelumnya, ini adalah spec dari komputer saya.

  • AMD Ryzen 7 5700G, 32GB RAM, RX6600
  • 4 Drive, 1 SSD untuk root (/) ext4, 3 HDD untuk penyimpanan ext4 juga

Sejauh ini, saya tahu ada 3 kategori general untuk mengatasi permasalahan ini

  1. Dual boot atau dual drive

Dual boot itu dengan menginstall 2 macam atau lebih sistem operasi dalam satu komputer. Biasanya dengan membagi satu disk menjadi beberapa partisi, lalu menginstall boot manager. Nantinya waktu menghidupkan komputer, ada pilihan mau masuk ke sistem operasi yang mana. Kalo dual drive, kita punya beberapa drive (disk) untuk setiap sistem operasi. Kalo mau boot ke sistem operasi tertentu, kita harus masuk ke BIOS, terus milih mau boot ke drive (disk) yang mana.

Bedanya apa?

Dual boot, lebih hemat. Cuman dengan 1 SSD misal, kita tinggal buat beberapa partisi. Dulu saya sering pakai cara ini waktu awal-awal pake linux. Problemnya selalu sama. Windows ngerusak boot manager, benerin nya repot. Kadang kalo salah waktu install, bisa kehapus satu partisi (saya pernah).

Dual drive, masalah biaya. Kita butuh dua atau lebih drive (disk) untuk setiap sistem operasi.

Kayaknya ga cocok dua-duanya buat kondisi saya sekarang. Yang pertama, saya cuman punya 1 SSD, sudah terinstall linux, data saya uda banyak. Dual boot harus mecah partisi, dan ga bisa mecah partisi tanpa ngehapus semua data. Yang kedua, saya lagi ga ada duit yang bisa dihambur-hamburkan cuman buat nge-mod Fallout 4. Dan SSD sekarang mahal cuy. HDD gimana? Gila ya, nginstall windows di HDD. HDD saya juga uda banyak data nya.

  1. Punya device sendiri untuk setiap sistem operasi

Buat beli SSD aja masih ga ada, gimana mau build pc khusus buat Windows?

  1. Virtualization

Jadi virtualization itu basically gini, kita menginstall sistem operasi diatas sistem operasi lain yang sedang berjalan. Untuk sistem operasi yang sedang berjalan, disebut host. Untuk sistem operasi yang diinstall secara virtual, disebut guest. Normalnya, sistem operasi yang diinstall secara virtual ini akan menggunakan perangkat virtual juga. Misal, CPU, GPU, hard disk, ethernet, semua virtual, bukan perangkat asli yang ada di komputer. Kelemahan cara ini adalah, karena perangkat nya virtual, kita jadi ga bisa maen Fallout 4 disana.

Tapi ada teknik yang dinamakan PCI Passthrough yang memungkinkan sistem operasi virtual untuk mengakses langsung perangkat PCI (dalam hal ini GPU atau VGA ya). Ini yang saya pilih. Jadi meski pake virtualization, saya bisa akses RX6600 dari dalam guest. Bisa buat maen Fallout 4.

Kekurangan dari cara ini, yang pertama, ribet. Dan dari pengalaman saya, apa yang bisa diaplikasikan sekarang, belum tentu bisa work terus-terusan. Soalnya sebenernya saya udah berulang kali pake cara ini. Awalnya bisa, tapi tiba-tiba suatu hari ga bisa dijalankan. Waktu itu ga tau penyebabnya dan males juga troubleshooting.

  Katanya sekarang manja, males utak-atik, la ini kan juga utak-atik

Mungkin wording nya yang salah. Saya males utak-atik buat hasil yang ga pasti. Utak-atik modding di linux, yang saya ga tau nanti hasil akhirnya bakal work atau enggak, yang kalo ga work ya berarti saya buang-buang waktu. Kalo utak-atik passthrough ini hasilnya jelas bisa work, konsepnya jelas, tinggal bener ga setting nya, gitu aja.

Yang kedua, namanya virtualisasi, kecuali komponen yang di passthrough, ga bisa pakai kemampuan komponen secara penuh. Contohnya, CPU dan RAM. Karena pada dasarnya kita menjalankan sistem operasi diatas sistem operasi lain yang sedang berjalan, maka alokasi penggunaan nya tidak bisa dimaksimalkan. CPU saya ada 16 logical CPU (8 cores 16 threads), yang saya alokasikan cuman 8, ga bisa seluruhnya karena dipakai juga sama host. RAM saya 32GB saya alokasikan 16GB, dengan alasan yang sama. Tapi cukup lah ya, masa 8cx16GB ga cukup buat main Fallout 4 (hint: terlalu cukup)

Dan di bawah, adalah cara saya untuk setup gpu passthrough dengan hasil yang paling konsisten.

Technicality

Untuk alasan kenapa saya menggunakan single gpu passthrough, scroll keatas atau gunakan link ini preambule

note: Karena saya menggunakan Archlinux, maka perintah yang digunakan pada pembahasan di bawah adalah perintah dari Archlinux.

disclaimer: Bagi yang mengikuti langkah-langkah pada pembahasan ini, segala sesuatu yang terjadi adalah hasil perbuatan dan tanggung jawab anda sendiri.


Enable Virtualization

Pertama, kita butuh mode UEFI di motherboard kita dan cek apakah CPU kita support virtualization (harusnya motherboard dan cpu modern sudah support 2 hal ini). Buat make sure, masuk ke BIOS dan aktifkan SVM/AMD-V untuk AMD CPU, atau Intel VT-x/VT-d untuk Intel CPU. Selain AMD/Intel saya ga tau ya

Set kernel parameter yang ada di grub (asumsi pake grub ya, kalo pake yang lain sesuaikan sendiri) untuk enable iommu

edit /etc/default/grub

amd cpu

GRUB_CMDLINE_LINUX_DEFAULT="... amd_iommu=on iommu=pt video=efifb:off ..."

intel cpu

GRUB_CMDLINE_LINUX_DEFAULT="... intel_iommu=on iommu=pt ..."

Kita update grub nya dengan

grub-mkconfig -o /boot/grub/grub.cfg

lalu restart/reboot


Check Hardware

Sekarang kita cek dulu IOMMU group dari GPU yang akan kita passthrough

shopt -s nullglob
for g in `find /sys/kernel/iommu_groups/* -maxdepth 0 -type d | sort -V`; do
    echo "IOMMU Group ${g##*/}:"
    for d in $g/devices/*; do
        echo -e "\t$(lspci -nns ${d##*/})"
    done;
done;

Copy paste script diatas ke terminal lalu enter. Kira-kira akan muncul seperti ini

IOMMU Group 0:
	00:01.0 Host bridge [0600]: Advanced Micro Devices, Inc. [AMD] Renoir PCIe Dummy Host Bridge [1022:1632]
IOMMU Group 1:
	00:01.1 PCI bridge [0604]: Advanced Micro Devices, Inc. [AMD] Renoir PCIe GPP Bridge [1022:1633]
IOMMU Group 10:
	02:00.0 PCI bridge [0604]: Advanced Micro Devices, Inc. [AMD/ATI] Navi 10 XL Downstream Port of PCI Express Switch [1002:1479]
IOMMU Group 11:
	03:00.0 VGA compatible controller [0300]: Advanced Micro Devices, Inc. [AMD/ATI] Navi 23 [Radeon RX 6600/6600 XT/6600M] [1002:73ff] (rev c7)
IOMMU Group 12:
	03:00.1 Audio device [0403]: Advanced Micro Devices, Inc. [AMD/ATI] Navi 21/23 HDMI/DP Audio Controller [1002:ab28]

Yang kita butuhkan nanti adalah VGA compatible controller. Apabila di dalam IOMMU Group tersebut ada lebih dari 1 komponen, maka nantinya kita harus memasukkan semua komponen yang ada dalam IOMMU Group tersebut. Kebetulan disini tidak ada komponen lain, jadi cukup masukkan satu komponen tersebut saja.

Perhatikan kode komponen VGA yang mau kita masukkan nantinya, kalau perlu catat (03:00.0)


Setup Virtual Machine

Sekarang saatnya kita install virtual machine nya

pacman -S qemu libvirt edk2-ovmf virt-manager dnsmasq ebtables

edit /etc/libvirt/libvirtd.conf

Hapus comment # dan sesuaikan seperti berikut

unix_sock_group = "libvirt"

unix_sock_rw_perms = "0770"

log_filters="3:qemu 1:libvirt"
log_outputs="2:file:/var/log/libvirt/libvirtd.log"

edit /etc/libvirt/qemu.conf

Hapus comment # dan sesuaikan seperti berikut (ganti user dengan nama user saat ini)

user = "user"

group = "user"

Tambahkan user ke group2 berikut (ganti user dengan nama user saat ini)

usermod -aG kvm,input,libvirt user

Setelah itu enable service nya

systemctl enable --now libvirtd
virsh net-start default
virsh net-autostart default

Installation Prep

Download virtio-win disini virtio-win.iso

Siapkan iso Windows yang bakal dipake

Langsung saja buka virt-manager (Virtual Machine Manager), buat baru, ingat-ingat namanya (usahakan tanpa spasi, misal “win10virt”), di akhir centang Customize before install

Di Overview, set Chipset ke Q35 dan firmware ke UEFI

Pada CPU, centang Configuration ke host-passthrough

Pada Memory, centang Enable shared memory, ini akan berguna nanti

Pada SATA Disk, pilih Disk bus ke VirtIO

Pada NIC, pilih Device model ke virtio

Add hardware, Storage, Select or create custom storage, Manage, pilih virtio-win.iso yang sudah didownload tadi, Device type nya CDROM device

Sekarang, klik Begin Installation, install Windows nya. Disini partisi drive nya ga keluar, maka Load driver dan pilih yang sesuai sama Windows yang lagi diinstall

Install Windows

Setelah Windows selesai diinstall, buka virtio, kebaca sebagai mount, install virtio win guest

Setelah itu shutdown, dan kita akan mulai utak-atiknya


Optional: Shared Drive

Kalo butuh shared drive, drive atau storage yang bisa diakses host dan guest, sebelum shutdown kita install WinFSP. Bisa di download disini.

Setelah diinstall, masuk ke Services, cari VirtIOFS Service, startup diset ke Automatic


ROM/vBIOS

Cek GPU

lspci -k | grep -EA3 'VGA|3D|Display'

Akan keluar kira-kira seperti ini

03:00.0 VGA compatible controller: Advanced Micro Devices, Inc. [AMD/ATI] Navi 23 [Radeon RX 6600/6600 XT/6600M] (rev c7)
	Subsystem: XFX Limited Device 6505
	Kernel driver in use: amdgpu
	Kernel modules: amdgpu

Disana kita bisa cari aja di google, misal “vbios xfx swft 210 rx 6600”

Atau, bisa kita ambil sendiri

Caranya

echo 1 > sudo tee /sys/bus/pci/devices/0000:03:00.0/rom
sudo cat /sys/bus/pci/devices/0000:03:00.0/rom > path/to/dump/vbios.rom
echo 0 > sudo tee /sys/bus/pci/devices/0000:03:00.0/rom

Sesuaikan kode device nya dengan device yang ada

Untuk AMD, kita ga perlu patch vbios yang ada

Tapi untuk NVIDIA, kita harus patch dulu vbios nya dengan menggunakan aplikasi hex editor seperti Okteta atau Ghex

Cari string “VIDEO”, dan hapus semua sebelum HEX value 55

Setelah itu simpan file vbios nya, ini nanti akan kita pakai

sudo mkdir /usr/share/vgabios
sudo cp ./vbios.rom /usr/share/vgabios
sudo chmod -R 644 /usr/share/vgabios/vbios.rom
sudo chown username:username /usr/share/vgabios.rom

Ganti username dengan nama user yang kita pakai


Scripting

Buat file executable untuk hooks qemu

mkdir /etc/libvirt/hooks
touch /etc/libvirt/hooks/qemu
chmod +x /etc/libvirt/hooks/qemu

Masukkan code ini ke dalam nya

#!/usr/bin/env bash

GUEST_NAME="$1"
HOOK_NAME="$2"
STATE_NAME="$3"
MISC="${@:4}"

BASEDIR="$(dirname $0)"

HOOKPATH="$BASEDIR/qemu.d/$GUEST_NAME/$HOOK_NAME/$STATE_NAME"

STDIN=$(cat)

set -ex # Jika script yang di load error, disini harus ikut error juga.
LOGFILE="/var/log/libvirt-hooks/qemu.log"

mkdir -p "$(dirname "$LOGFILE")"
echo "[HOOK DISPATCH] $(date): $@" >> "$LOGFILE"

# cek apa script yang nanti kita buat exist
if [ -f "$HOOKPATH" ] && [ -s "$HOOKPATH" ] && [ -x "$HOOKPATH" ]; then
    eval \"$HOOKPATH\" "$@" <<< "$STDIN" # Memanggil script sesuai path
elif [ -d "$HOOKPATH" ]; then
    while read file; do
        # check for null string
        if [ ! -z "$file" ]; then
          eval \"$file\" "$@" <<< "$STDIN"
        fi
    done <<< "$(find -L "$HOOKPATH" -maxdepth 1 -type f -executable -print;)"
fi

credit to SharkWipf

Jadi script ini fungsinya adalah hook, yang akan dijalankan oleh qemu waktu kita menghidupkan virtual machine (Windows). Setelah selesai menulis script diatas, kita restart dulu libvirt nya buat make sure script nya ke load.

systemctl restart libvirtd

Setelah itu coba jalankan virtual machine nya, lalu shutdown lagi buat ngetest aja

Kita bisa buka /var/log/libvirt-hooks/qemu.log maka akan keluar log dengan awalan [HOOK DISPATCH] seperti yang ada di script

[HOOK DISPATCH] Fri May 22 18:41:12 WIB 2026: win10virt prepare begin -
[HOOK DISPATCH] Fri May 22 18:41:18 WIB 2026: win10virt start begin -
[HOOK DISPATCH] Fri May 22 18:41:26 WIB 2026: win10virt started begin -
[HOOK DISPATCH] Fri May 22 18:51:06 WIB 2026: win10virt stopped end -
[HOOK DISPATCH] Fri May 22 18:51:06 WIB 2026: win10virt release end shutdown

Tanda nya script kita uda jalan

Di log bisa kita lihat bahwa saat kita menjalankan virtual machine, ada beberapa tahap proses. Prepare, Start, Started, Stopped, Release. Dari sana lah nanti kita akan buat script untuk mem-passthrough hardware kita (dalam kasus ini GPU). Kita perlu persiapkan di fase Prepare dan Release.

Kita harus persiapkan hardware nya agar bisa diakses oleh virtual machine. Hardware itu ga bisa diakses bareng, jadi kita harus diputus dulu akses komputer kita ke hardware. Proses ini dilakukan waktu fase prepare di virtual machine. Kita buat script nya

mkdir -p /etc/libvirt/hooks/qemu.d/win10virt/prepare/begin
touch /etc/libvirt/hooks/qemu.d/win10virt/prepare/begin/start.sh
chmod +x /etc/libvirt/hooks/qemu.d/win10virt/prepare/begin/start.sh

ganti win10virt sesuai nama virtual machine yang kita buat tadi

edit start.sh

LOG="/var/log/libvirt-hooks/start.log"
exec >> "$LOG" 2>&1

echo "$(date '+%Y-%m-%d %H:%M:%S') hook called: $0 $@"

set -x

systemctl stop display-manager # pakai ini kalo gnome
# systemctl --user -M YOUR_USERNAME@ stop plasma* # pakai ini kalo plasma

while systemctl is-active --quiet display-manager; do
    sleep 1
done
sleep 2  # buffer buat nungguin GPU detach

echo 0 > /sys/class/vtconsole/vtcon0/bind
echo 0 > /sys/class/vtconsole/vtcon1/bind

# Unbind EFI Framebuffer
echo efi-framebuffer.0 > /sys/bus/platform/drivers/efi-framebuffer/unbind
sleep 2
# Unload NVIDIA kernel modules
# modprobe -r nvidia_drm nvidia_modeset nvidia_uvm nvidia

# Unload AMD kernel module
modprobe -r amdgpu

# Detach GPU devices dari host
# Pake kode GPU atau HDMI Audio PCI host device
virsh nodedev-detach pci_0000_03_00_0
virsh nodedev-detach pci_0000_03_00_1

# Load vfio module
modprobe vfio-pci

Lalu kita buat kebalikannya untuk ngembalikan hardware yang dipassthrough biar bisa dipake sama host

mkdir -p /etc/libvirt/hooks/qemu.d/win10virt/release/end
touch /etc/libvirt/hooks/qemu.d/win10virt/release/end/stop.sh
chmod +x /etc/libvirt/hooks/qemu.d/win10virt/release/end/stop.sh

edit stop.sh

LOG="/var/log/libvirt-hooks/stop.log"
exec >> "$LOG" 2>&1

echo "$(date '+%Y-%m-%d %H:%M:%S') hook called: $0 $@"

set -x

# Attach GPU devices ke host host
# kembalikan GPU atau HDMI Audio PCI host device
virsh nodedev-reattach pci_0000_03_00_0
virsh nodedev-reattach pci_0000_03_00_1

# Unload vfio module
modprobe -r vfio-pci

# Load AMD kernel module
modprobe amdgpu

# Rebind framebuffer ke host
echo "efi-framebuffer.0" > /sys/bus/platform/drivers/efi-framebuffer/bind

# Load NVIDIA kernel modules
# modprobe nvidia_drm
# modprobe nvidia_modeset
# modprobe nvidia_uvm
# modprobe nvidia

# Bind VTconsoles
echo 1 > /sys/class/vtconsole/vtcon0/bind
echo 1 > /sys/class/vtconsole/vtcon1/bind

# Restart Display Manager
systemctl start display-manager

Sekarang kita mulai hapus dan tambah hardware nya

Buka virtual machine kita (bukan dijalankan, buka aja config nya), lalu hapus Channel Spice, Display Spice, Video QXL, Sound ich*, SATA CDROM

Jangan dijalankan dulu apapun yang terjadi

Tambahkan hardware (Add Hardware)

  1. PCI Host Device, GPU/VGA yang akan di passthrough. Kalo dalam IOMMU Group nya ada device lain, masukkan juga
  2. PCI Host Device, HD Audio Controller. Ini untuk sound ya. Kalo misal pake headset usb atau koneksi lainnya, nanti masukkan juga.
  3. USB Device, ini masukkan mouse, keyboard, atau perangkat lain yang nantinya mau dipake di Windows. Misal pake bluetooth dongle, itu dimasukkan juga.
  4. Optional, shared drive. Filesystem Passthrough, Driver virtiofs, Source path masukkan lokasi yang mau dishare, Target path isi nama Drive yang mau dimunculkan. Misal, namanya “shared_drive”, maka nanti waktu masuk ke Windows, akan muncul drive Z: dengan nama Shared Drive. Nah disini fungsi nya “Enable shared memory” di bagian RAM/Memory. Kalo di Memory belum diset, set sekalian.

Sekarang, kita buka bagian GPU/VGA. Kita perlu menambahkan vBIOS yang sudah kita siapkan tadi. Buka tab XML lalu ubah kira-kira menjadi seperti ini

<hostdev mode="subsystem" type="pci" managed="yes">
  <source>
    <address domain="0x0000" bus="0x03" slot="0x00" function="0x0"/>
  </source>
  <rom file="/usr/share/vgabios/vbios.rom"/>
  <address type="pci" domain="0x0000" bus="0x06" slot="0x00" function="0x0"/>
</hostdev>

Tambahkan lokasi vbios nya di antara <source> dan <address>, apply.
